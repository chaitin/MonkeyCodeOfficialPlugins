# 企业官网开发（Corporate Website Dev）

## 使用场景

仅当用户需要**从零开始**构建全新的企业/产品官网或品牌站时使用当前技能。

## 注意 

- 需求必须覆盖完整链路：调研、项目文档、编码实现、部署测试，且包含多页面、UI 风格、官方色彩等品牌要求。
- 启用本技能时必须**同步加载 `ui-ux-pro-max` 技能**，所有 UI 设计方案要遵循 `ui-ux-pro-max` 技能提供的设计思路与流程。
- 启用本技能时必须**同步加载 `bootstrap-helper` 技能**，前端实现强制采用 **Vite + Bootstrap** 组合，不允许更换框架或样式体系；若用户提出不同技术诉求，需要先确认是否仍适用本技能。

## 工作流程（四步）

### 1. 需求调研

- 至少需要了解到一下信息：
  1. 业务所在的行业（单选，金融、教育、医疗保健、科技、游戏、影视文化、新闻传媒等）
  1. 业务背景（自由文本）。
  2. 需要的页面（多选，允许自定义，包含但不限于：首页、关于我们、产品展示、客户案例、解决方案等）。
  3. 业务重点（突出核心产品、突出服务能力、突出产品演示、故事驱动）
  3. 明亮模式还是暗黑模型。
  4. UI 特色（多选，包括但不限于：玻璃态、扁平化、拟物化、像素风、极简风、赛博朋克等）。
- 提问应该尽量使用内置的提问工具。
- 所有必答项信息完整且无歧义后才能进入下一阶段；否则需要继续提问澄清。

### 2. 编写项目文档

- 将调研结果整理成项目需求书，写到 `project.md` 中 
- 项目需求书需要经过用户的审查确认，必要时迭代更新；若 UI 方案与 `ui-ux-pro-max` 原则冲突，需回到调研阶段修正。

### 3. 开发编码（固定栈：Vite + Bootstrap）

- 前端实现强制采用 **Vite + Bootstrap** 组合，不允许更换框架或样式体系；若用户提出不同技术诉求，需要先确认是否仍适用本技能。
- 根据 `bootstrap-help` 的指导来创建项目并进行开发
- 所有组件和交互方案需对照 `ui-ux-pro-max` 的设计模式（信息密度、留白、动效）。
